SOLID Principles Applied

S — Single Responsibility

One class, one job

Each class has exactly one reason to change.

UserController    → authentication only
HotelController   → hotel search only
BookingController → reservations only
PriceCalculator   → pricing only
User              → guest data only
O — Open/Closed

Open for extension, closed for modification

Room is abstract. Adding a PentHouseRoom requires zero changes to existing code:

class PentHouseRoom extends Room {
    protected double getPriceMultiplier() { return 8.0; }
    public String getRoomType() { return "Penthouse"; }
}
L — Liskov Substitution

Subtypes must be substitutable

Any code using Room works identically with StandardRoom, DeluxeRoom, or SuiteRoom:

Room r = RoomFactory.createRoom(type, ...);
r.getFinalPrice();    // works for ALL subtypes
r.getRoomType();      // works for ALL subtypes
I — Interface Segregation

No unused methods in interfaces

IView contains only what a view needs — nothing more. Controllers don't depend on methods they don't use.

D — Dependency Inversion

Depend on abstractions, not concretions

// WRONG
BookingController ctrl = new BookingController(new ConsoleView());

// RIGHT — inject abstraction
IView view = new ConsoleView();
BookingController ctrl = new BookingController(view, hotelCtrl);

Swapping ConsoleView for a WebView requires changing only Main.java.

Creational Design Patterns

Factory Method

RoomFactory.java

Creates Room objects based on runtime type selection. Calling code never uses new DeluxeRoom() directly.

BookingController
→
RoomFactory.createRoom(DELUXE)
→
new DeluxeRoom()
Room room = RoomFactory.createRoom(
    RoomFactory.RoomType.SUITE,
    roomId, hotelId, guestName, roomNumber, basePrice
);
Builder Pattern

BookingBuilder.java

Constructs a Booking step by step. Avoids unreadable constructors with 10+ parameters.

Booking booking = new BookingBuilder(bookingId, userId)
    .hotel(hotel)
    .rooms(rooms)
    .nights(3)
    .checkIn("2025-06-01")
    .contactEmail("guest@email.com")
    .paymentStatus("CONFIRMED")
    .build();
Singleton Pattern

SystemConfig.java

Guarantees one shared instance of system configuration across the entire application. Thread-safe.

SystemConfig config = SystemConfig.getInstance();
double price  = config.getBasePricePerNight(); // ₹1500
double gst    = config.getGstRate();           // 0.18
int    maxBkg = config.getMaxBookingsPerUser(); // 5
Prototype Pattern

HotelPrototype.java

Hotel templates are registered once and cloned with updated availability as needed.

// Template registered once
registry.put("TAJ", new Hotel("H001", "Taj Palace", ...));

// Cloned with today's availability
Hotel h = prototype.getHotelTemplate("TAJ", 45);
